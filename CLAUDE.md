# CLAUDE.md — AT App Workspace

This workspace contains the Alexander Technique reminder app and its general public flavor variant.

---

## What This Is

A cross-platform mobile app (iOS, Android, web — built in Flutter) that delivers awareness prompts throughout the day via timed notifications. Works on a locked/sleeping phone.

**Two flavors, built sequentially:**
1. **AT Flavor** *(build this first)* — for Alexander Technique teachers and practitioners. Uses AT-specific language (inhibition, direction, primary control, primary control sequence, etc.)
2. **General Public Flavor** *(post-AT v1)* — same engine, less AT-specific language. Broader somatic/posture awareness audience.

The codebase is flavor-ready from day one. Flavor controls: app name, bundle ID, icon, color scheme, and default prompt library.

## App Overview

**Delivery modes:**
- **Free Mode** — prompts fire at random or fixed intervals from the active library throughout the day
- **Sequence Mode** — ordered set of prompts fires in rapid succession (~2 sec gaps), triggered on demand or on a repeating timer

**Audio modes:** Silent | Tone | Voice (OS TTS) | Tone + Voice (Pavlov conditioning: train with tone+voice → switch to tone only)

**Key features:**
- Prompt libraries: built-in AT sets (All Prompts 200, Bruce's 23, MIO 9) + custom user libraries
- Blackout windows — block prompts by day/time (e.g. massage therapy session hours)
- Random or sequential prompt order
- Live countdown on main screen, manual trigger button
- Sequence builder — create custom ordered sequences

---

## Key Architecture Constraints

- **No backend at v1** — fully local, no auth, no network required
- **Background execution on iOS is the hardest problem** — validate on physical device early; simulators are unreliable
- **Flavor architecture from day one** — structural changes must remain flavor-compatible
- **Dark mode default** across both flavors

## v3 Future Explorations (not now)

**Desktop app with reactive posture detection** — inspired by Dorso (Mac app). Fires prompts when poor use is detected rather than on a fixed timer.

- **Mac**: Apple Vision framework (`VNDetectHumanBodyPoseRequest`) — face position in frame, head pose, shoulder/ear alignment. Processes on-device, no frames sent anywhere.
- **AirPods**: `CMHeadphoneMotionManager` (CoreMotion) — detects head pitch to ~1-2° resolution. Forward head carriage shows clearly in pitch data.
- **Cross-platform**: MediaPipe (Google, open source, local) — same capabilities on Windows and Linux. Flutter has desktop support for all three platforms.
- **Screen blur vs. prompt overlay** — blur (Dorso approach) is a passive interruption; AT prompt overlay may be more philosophy-aligned (inhibition before direction)
- **Feature concepts are not copyrightable** — no legal concern building similar functionality

Desktop version will look significantly different from the phone app but share the same prompt libraries, blackout windows, and scheduling logic.
