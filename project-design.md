# Alexander Technique (AT) — Design Document
> Generated by /project-designer. Ready for build agent.

---

## 1. Overview

A cross-platform mobile app (iOS, Android, web) built in Flutter that delivers Alexander Technique awareness prompts throughout the day via timed notifications. The app functions as an intelligent, customizable prompt timer — firing AT directions at random or fixed intervals, with optional voice or chime audio, and full control over scheduling, libraries, and sequences. Built for the developer's personal use first, then the broader AT community. Codebase is flavor-ready for a second general-public app variant from day one.

---

## 2. Goals

- **Build a reliable background prompt timer** that works on a locked/sleeping phone
- **Replace the existing Scriptable + Apple Reminders workaround** with a dedicated, polished app
- **Success at launch:** App runs on iPhone and browser. Prompts fire reliably in background. User can configure interval, library, audio mode, and blackout windows. No jitter, no bugs.
- **Success at 6 months:** App is on the App Store. AT teachers and students are using it. The Pavlov tone-conditioning flow is working as intended. A second flavor (general public) is in progress.

## 5. Core Features (MVP)

### Delivery Modes

| Feature | Description |
|---|---|
| **Free Mode** | Prompts fire throughout the day at random or fixed intervals. Any prompt from the active library, random or sequential. Fully configurable frequency. Blackout windows respected. |
| **Sequence Mode** | A defined ordered set of prompts fires in rapid succession (~2 sec gap). Can be triggered on-demand (tap) OR on a repeating timer (e.g. every hour). Under 10 prompts per sequence. Built-in sequences included; user can create custom sequences. |

### Prompt Libraries

| Feature | Description |
|---|---|
| **All Prompts library** | Full 200-prompt AT set, built-in |
| **Bruce's Prompts** | 23 prompts, separate selectable library |
| **MIO Prompts** | 9 minimalist prompts, separate selectable library |
| **Custom libraries** | User creates named libraries, adds/removes prompts |
| **Mix and match** | User selects multiple active libraries; prompts pool together |

### Prompt Management

| Feature | Description |
|---|---|
| **Edit prompts** | Edit text of any prompt |
| **Reorder prompts** | Drag to reorder within a library |
| **Add / delete prompts** | Full CRUD on custom prompts |
| **Sequence builder** | Create sequences, set prompt order, set gap duration |

### Audio

| Mode | Behaviour |
|---|---|
| 🔕 **Silent** | Notification banner + vibration only |
| 🔔 **Tone** | Chime/bell plays, no voice |
| 🗣️ **Voice** | OS TTS speaks the prompt aloud |
| 🔔🗣️ **Tone + Voice** | Chime plays → then voice speaks (Pavlov training mode) |

- OS built-in voices via `flutter_tts` — user selects from available device voices
- Adjustable speech rate and pitch
- Multiple built-in chime options (soft bell, Tibetan bowl, simple tone — minimum 3)
- Pavlov logic: train with Tone + Voice → condition the association → switch to Tone only

### Scheduling & Blocking

| Feature | Description |
|---|---|
| **Fixed interval** | Prompt fires every N minutes |
| **Random interval** | Prompt fires at random gaps within a user-set min/max range |
| **Random prompt** | Each prompt fires randomly from the active library |
| **Sequential prompt** | Prompts fire in library order, cycling |
| **Blackout windows** | User defines time blocks by day of week where no prompts fire. Multiple windows per day supported. |

### Main Screen

- Active mode and library displayed
- Live countdown to next prompt or sequence
- Manual trigger button (fires a prompt immediately)
- Start / stop toggle
- Visual animation: screen breathes/pulses when a prompt fires; prompt text fades in, holds, fades out
- Ambient subtle animation between prompts when app is open

### Settings Screen (hub)

- Delivery mode selection (Free / Sequence)
- Interval settings (fixed or random, min/max range)
- Random vs sequential prompt toggle
- Active library selection (multi-select)
- Sequence configuration (trigger frequency, which sequence)
- Audio mode selection
- Voice selection and speech rate/pitch
- Chime selection
- Link to: Blackout Windows manager, Library manager, Sequence manager

## 6. Technical Architecture

```
┌─────────────────────────────────────────────┐
│               Flutter App                   │
│  ┌──────────┐  ┌──────────┐  ┌───────────┐  │
│  │   iOS    │  │ Android  │  │    Web    │  │
│  └────┬─────┘  └────┬─────┘  └─────┬─────┘  │
│       └─────────────┴───────────────┘        │
│            Single Dart codebase              │
│    (shared UI + business logic + themes)     │
└──────────────────┬──────────────────────────┘
                   │
         ┌─────────▼──────────┐
         │   Local Storage    │
         │  (Hive / Isar)     │
         │  Prompts, libs,    │
         │  settings, windows │
         └────────────────────┘

Background Execution:
  iOS  → flutter_background_service + AVAudioSession
  Android → WorkManager or foreground service
  Web  → Limited (tab must be open); primary use is mobile
```

### Key Architectural Decisions

- **Flavor architecture from day one** — app name, bundle ID, icon, color scheme, and default library are flavor-controlled. Second flavor (general public) slots in without structural changes.
- **State management: Riverpod** — consistent with developer's existing Flutter codebase (task manager app)
- **Local-only storage** — no backend, no auth, no network required. All data lives on device.
- **Background execution** — the hardest technical problem. Requires native configuration on iOS (audio session + background modes). Must be validated early in the build.
- **Audio session** — configured as ambient/playback so TTS and chimes fire even on locked screen and in silent mode (using `flutter_tts` + `just_audio` or `audioplayers`)

## 7. Tech Stack

| Layer | Choice | Rationale |
|---|---|---|
| UI framework | Flutter (Dart) | Single codebase for iOS, Android, web; developer has existing experience |
| State management | Riverpod | Consistent with developer's task manager app |
| Local storage | Hive or Isar | Fast, offline-first, no network; suited for prompt/settings data |
| TTS | flutter_tts | Wraps iOS AVSpeechSynthesizer + Android TTS; free, offline, voice-selectable |
| Audio playback | just_audio or audioplayers | Plays chime files; background audio capable |
| Background execution | flutter_background_service | Cross-platform background timer; iOS requires native entitlements |
| Notifications | flutter_local_notifications | Scheduled local notifications with prompt text |
| Animations | Flutter built-in (AnimationController, TweenAnimationBuilder) | Breathing/pulse/fade effects on main screen |
| Flavor system | Flutter flavors (--flavor flag) | Separate bundle IDs, names, icons, themes per flavor |

## 8. Data Model

All data stored locally. No user ID needed at v1.

### `Prompt`
```
- id: String (UUID)
- text: String
- libraryId: String
- sortOrder: int
- isBuiltIn: bool
- createdAt: DateTime
```

### `Library`
```
- id: String (UUID)
- name: String
- isBuiltIn: bool         ← All Prompts, Bruce's Prompts, MIO Prompts
- sortOrder: int
- createdAt: DateTime
```

### `Sequence`
```
- id: String (UUID)
- name: String
- promptIds: List<String>  ← ordered list
- gapSeconds: int          ← default 2
- isBuiltIn: bool
- createdAt: DateTime
```

### Built-in sequences (seed data)
1. **Primary Control** — neck free → head forward and up → back lengthen and widen
2. **Body Scan** — head to toe awareness sweep
3. **Pre-activity** — quick set before sitting/standing/lifting

### `BlackoutWindow`
```
- id: String (UUID)
- label: String            ← e.g. "Massage Block"
- daysOfWeek: List<int>    ← 1=Mon ... 7=Sun
- startTime: String        ← "HH:mm"
- endTime: String          ← "HH:mm"
```

### `AppSettings`
```
- deliveryMode: enum (free, sequence)
- intervalType: enum (fixed, random)
- fixedIntervalMinutes: int
- minIntervalMinutes: int
- maxIntervalMinutes: int
- promptOrder: enum (random, sequential)
- activeLibraryIds: List<String>
- audioMode: enum (silent, tone, voice, toneAndVoice)
- selectedVoiceName: String
- speechRate: double
- speechPitch: double
- selectedChime: String    ← asset key
- sequenceMode: enum (onDemand, timer)
- sequenceTimerMinutes: int
- activeSequenceId: String?
- isRunning: bool
- themeMode: enum (dark, light, system)  ← dark default
```

### `AppFlavor` (build-time constant, not stored)
```
- flavorName: String       ← "at" | "general"
- appDisplayName: String   ← "Alexander Technique" | TBD
- bundleId: String
- primaryColor: Color
- defaultLibraryId: String
```

## 13. Deployment & Infrastructure

### Development / Testing
- Run on physical iPhone via Xcode (free provisioning profile)
- Run on Android via USB/ADB
- Run in browser via `flutter run -d chrome` (limited background functionality)
- **Background execution must be validated on physical device early** — simulators do not accurately replicate background behavior

### Production
- **iOS:** App Store (requires Apple Developer account, $99/yr)
- **Android:** Google Play Store
- **Web:** Firebase Hosting or similar (secondary platform — primarily for testing)

### Flavor Builds
```
flutter run --flavor at -t lib/main_at.dart         # AT flavor
flutter build ios --flavor at -t lib/main_at.dart   # AT iOS release
flutter build apk --flavor at -t lib/main_at.dart   # AT Android release
```

### Environments
- Single environment at v1 (no staging/prod split needed for local-only app)

## 14. Testing Strategy

| Type | Approach |
|---|---|
| **Manual on device** | Primary approach — test background timer behavior on physical iPhone |
| **Unit tests** | Background timer logic, blackout window evaluation, random interval generation |
| **Widget tests** | Main screen animation states, settings form |
| **Integration tests** | Skip at v1 |

**Highest-risk component:** Background execution on iOS. Must be tested on a physical device from day one — not in simulator. Validate that:
- Prompts fire when app is in background
- Prompts fire when screen is locked
- Audio plays on locked screen
- Timer survives app being backgrounded for 30+ minutes

## 15. Open Questions

1. **Sequence prompt content** — Built-in sequence prompts (Primary Control, Body Scan, Pre-activity) need specific prompt text defined before build. Developer to provide before implementation of Sequence Mode begins.

2. **Library categories** — The prompt spreadsheet has 4 planned categories (Lying/Sitting/Walking/General) but these are not yet assigned. Treat as v2 — launch with uncategorized All Prompts library.

3. **App name — final decision** — Working name is "Alexander Technique." Final name (and whether a trademark check is needed) TBD before App Store submission.

4. **Blackout window times** — Developer's massage therapy schedule times need to be confirmed for seed data / onboarding defaults.

5. **Design spec** — Visual design (color palette, typography, animation behavior, component styling) to be defined in a separate `design-spec.md` before build begins. Builder should use dark mode with pill buttons and open/spacious layout as baseline defaults until spec is delivered.

6. **Chime audio files** — 3+ chime options needed (soft bell, Tibetan bowl, simple tone). Source/record before audio implementation begins.

7. **Web background limitation** — Web cannot reliably run background timers (tab must stay open). Decide whether to include a visible disclaimer in the web version or deprioritize web to a "preview/test" platform only.

## 16. Assumptions Made

| Assumption | Reasoning |
|---|---|
| Flutter over React Native | Developer has existing Flutter experience from task manager app |
| Riverpod for state management | Consistent with existing codebase |
| Local-only storage (no backend) | v1 is single-user, no sync needed; keeps architecture simple and free |
| Hive or Isar for local DB | Better suited than SQLite for document-style data at this scale |
| Background execution via flutter_background_service | Most reliable cross-platform option; requires native iOS entitlements |
| AT flavor built first | Developer is the primary test user |
| Second flavor deferred but architecture supports it from day one | Flavor system configured in build scripts; ThemeData and copy are flavor-injected |
| Prompt categories deferred to v2 | Category data not yet assigned in source spreadsheet |
| No App Store publishing of task manager was fine; this app IS intended for App Store | Different distribution strategy confirmed |
| Dark mode default | Developer confirmed preference |
| 200 prompts from spreadsheet (All_Prompts sheet) as canonical seed data | Confirmed as the master list |
| Bruce's Prompts (23) and MIO Prompts (9) as separate built-in libraries | Confirmed |
| Sequences fire prompts ~2 seconds apart | Developer specified; not user-adjustable at v1 |
| Web platform included primarily for testing | Background timer limitations make mobile the primary platform |
